# Load & Performance Testing

Parse `$ARGUMENTS[0]` as the sub-command (`run` | `bundle` | `baseline`, default: `run`).
Parse `$ARGUMENTS[1]` as the target (endpoint path or script file).

---

## `/test-load run <endpoint>`

### Step 1 — Pre-flight

```bash
# Check k6 is installed
k6 version 2>/dev/null || echo "NOT INSTALLED"

# Detect staging base URL from SSM or .env
aws ssm get-parameter --name "/<project>/staging/domain" --query Parameter.Value --output text 2>/dev/null \
  || grep 'VITE_API_URL\|API_URL' .env.staging 2>/dev/null | head -1
```

If k6 is not installed:
> k6 is not installed. Install with:
> - macOS: `brew install k6`
> - Docker: `docker run --rm -i grafana/k6 run -`
>
> Install k6 then re-run `/test-load`.

### Step 2 — Find or generate a k6 script

```bash
find . -path '*/load-tests/*.js' -o -path '*/k6/*.js' 2>/dev/null | head -10
```

If a script exists for the endpoint: use it.
If not: generate `load-tests/<endpoint-slug>.js`:

```js
// load-tests/<endpoint-slug>.js
// Generated by /test-load — edit thresholds to match your SLOs
import http from 'k6/http'
import { check, sleep } from 'k6'
import { Rate, Trend } from 'k6/metrics'

const errorRate = new Rate('errors')
const duration = new Trend('request_duration')

export const options = {
  stages: [
    { duration: '30s', target: 10 },   // ramp up
    { duration: '1m',  target: 50 },   // sustain load
    { duration: '30s', target: 100 },  // peak
    { duration: '30s', target: 0 },    // ramp down
  ],
  thresholds: {
    http_req_duration: ['p(50)<200', 'p(95)<500', 'p(99)<1000'],
    http_req_failed:   ['rate<0.01'],   // <1% errors
    errors:            ['rate<0.01'],
  },
}

const BASE_URL = __ENV.BASE_URL || 'https://api-staging.<domain>'

export function setup() {
  // Authenticate once — reuse token across VUs
  const res = http.post(`${BASE_URL}/api/v1/auth/login`, JSON.stringify({
    username: __ENV.TEST_USERNAME,
    password: __ENV.TEST_PASSWORD,
  }), { headers: { 'Content-Type': 'application/json' } })
  return { token: res.json('token') }
}

export default function(data) {
  const headers = {
    'Authorization': `Bearer ${data.token}`,
    'Content-Type': 'application/json',
  }

  const res = http.get(`${BASE_URL}<ENDPOINT>`, { headers })

  const ok = check(res, {
    'status 200':        r => r.status === 200,
    'response < 500ms':  r => r.timings.duration < 500,
    'body not empty':    r => r.body.length > 0,
  })

  errorRate.add(!ok)
  duration.add(res.timings.duration)
  sleep(1)
}
```

Ask the user: does this script need an auth token? What test credentials to use?
If `TEST_USERNAME`/`TEST_PASSWORD` are needed, check `.env.staging` — do not generate credentials.

### Step 3 — Run the test

```bash
BASE_URL=<staging-url> \
TEST_USERNAME=<from-env> \
TEST_PASSWORD=<from-env> \
k6 run --out json=load-test-results.json load-tests/<slug>.js
```

Show live output. After completion, parse results:

```bash
cat load-test-results.json | k6 inspect 2>/dev/null \
  || node -e "
const lines = require('fs').readFileSync('load-test-results.json','utf8').trim().split('\n')
const metrics = {}
lines.forEach(l => {
  try {
    const d = JSON.parse(l)
    if (d.type === 'Point' && d.metric) {
      metrics[d.metric] = metrics[d.metric] || []
      metrics[d.metric].push(d.data.value)
    }
  } catch(e){}
})
const pct = (arr, p) => { arr.sort((a,b)=>a-b); return arr[Math.floor(arr.length*p/100)] }
const dur = metrics['http_req_duration'] || []
console.log('p50:', pct(dur,50).toFixed(0)+'ms')
console.log('p95:', pct(dur,95).toFixed(0)+'ms')
console.log('p99:', pct(dur,99).toFixed(0)+'ms')
console.log('errors:', (metrics['http_req_failed']||[]).filter(v=>v>0).length, '/')
"
```

### Step 4 — Compare against baseline

```bash
find docs/perf/ -name 'BASELINE-*.md' 2>/dev/null | sort -r | head -1
```

If a baseline exists: compare each metric and show delta (▲ better / ▼ worse).

Write to `docs/perf/LOAD-<date>-<slug>.md`:


## `/test-load bundle`

Check frontend bundle size against a size budget.

```bash
# Build and analyse
npm run build 2>&1 | tail -20

# Check with bundlesize if configured
npx bundlesize 2>/dev/null || echo "bundlesize not configured"

# Manual size check
find dist/ -name '*.js' -not -name '*.map' 2>/dev/null \
  | xargs ls -lh | sort -k5 -rh | head -10

# Check for large dependencies
npx vite-bundle-visualizer 2>/dev/null || \
  npx source-map-explorer 'dist/assets/*.js' 2>/dev/null || \
  echo "Install vite-bundle-visualizer: npm i -D vite-bundle-visualizer"
```

Report:

> **Bundle size report**
>
> | Chunk | Size | Gzipped | Budget | Status |
> |---|---|---|---|---|
> | main.js | 342 KB | 98 KB | 150 KB gzip | ❌ Over budget |
> | vendor.js | 1.1 MB | 312 KB | 400 KB gzip | ✅ |
>
> Largest dependencies: `lodash` (71KB), `moment` (68KB)
> → Consider: `lodash-es` tree-shaking, `date-fns` instead of `moment`

Write `bundlesize` config if not present:

```json
// package.json (add this section)
"bundlesize": [
  { "path": "./dist/assets/index-*.js", "maxSize": "150 kB" },
  { "path": "./dist/assets/vendor-*.js", "maxSize": "400 kB" }
]
```
